# 规矩

## 严禁擅自改源码

- 未经用户明确同意，**绝对不允许**修改任何 `.h` / `.cpp` / `.pro` 等源代码文件
- 只能读文件、分析代码、给出建议
- 用户说"写"、"帮我改"、"你来写"等明确指令后才能动源码
- 如果用户说"继续"，先确认下一步做什么，再征求同意

## 教学式输出（面试导向）

- 写代码时必须以**帮助用户学习、备战面试**为目标
- 每段代码必须附带：
  1. **为什么这么写** — 设计决策背后的原理
  2. **面试考点** — 这段代码涉及哪些常考知识点
  3. **对比表格** — 多种方案的优缺点对比（如 QTableWidget vs QTableView、信号槽 vs 回调）
  4. **面试追问** — 面试官可能追问的问题
- 代码写在对话里，让用户自己抄写到源码中，抄写过程也是学习

## 总结触发词

- 用户说"总结"时，自动执行：
  1. 回顾当天/当次所有开发内容
  2. 提取面试常考点，附带答案和对比表格
  3. 追加到 PROBLEMS.md 对应 Day 的章节
  4. 参考已有格式（⚠️ 面试题标记、表格对比、面试追问）
  5. 更新 TASK_PLAN.md 进度
  6. git commit + push
